# Remotion 故障排查指南

## 常见错误及解决方案

### 1. 编译/构建错误

**症状**：构建过程中出现语法错误或类型错误
**排查步骤**：
1. 检查语法是否正确，特别注意括号匹配和分号
2. 确认类型定义是否完整
3. 检查依赖版本是否兼容
4. 清除缓存后重新构建

### 2. 运行时错误

**症状**：应用运行时抛出异常
**排查步骤**：
1. 查看错误堆栈，定位出错位置
2. 检查变量是否为 null/undefined
3. 确认 API 调用是否正确
4. 检查异步操作是否正确处理

### 3. 性能问题

**症状**：页面加载缓慢或交互卡顿
**排查步骤**：
1. 使用 Performance 面板分析瓶颈
2. 检查是否有不必要的重渲染
3. 分析网络请求是否过多或过大
4. 检查是否有内存泄漏

### 4. 样式问题

**症状**：样式不生效或布局异常
**排查步骤**：
1. 检查选择器优先级
2. 确认 CSS 属性是否被覆盖
3. 检查盒模型和布局方式
4. 验证响应式断点设置

## 调试工具

| 工具 | 用途 | 使用场景 |
|------|------|---------|
| DevTools | 浏览器调试 | 性能分析、DOM检查、网络请求 |
| Console | 日志输出 | 变量检查、错误追踪 |
| Profiler | 性能分析 | 渲染性能、内存使用 |
| Network | 网络分析 | 请求时序、资源大小 |
